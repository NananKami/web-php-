# 【CTF/Web安全学习笔记】动态代码注入与“无文件”木马 (RCE)---2026.2.25学习笔记

## 📌 漏洞原理：什么是代码注入？

在许多早期的 Web 架构或存在缺陷的代码中，服务器会将用户输入的参数直接作为代码执行。

与**文件上传漏洞**（需要物理上传一个 `.php` 文件到服务器硬盘）不同，**代码注入**是“无文件”的。攻击者直接通过 URL 或表单，将恶意指令送到服务器的内存中动态解析。

这在底层的逻辑漏洞上，就如同在 **C 语言**中，将一个未经任何清洗和过滤的字符指针直接传给了 `system()` 函数，导致系统直接执行了外部输入的任意指令，从而造成严重的执行流劫持。

## 💻 经典靶场场景复盘
**漏洞代码示例：**
```php
<?php
    $a = $_GET['hello']; // 接收 URL 中的 GET 参数
    eval($a);            // eval() 函数将字符串作为 PHP 代码无条件执行
?>
